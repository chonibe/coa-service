## Summary

The README describes COA Service as a comprehensive digital art authentication and management platform. It is built to issue NFC-backed Certificates of Authenticity for artworks sold through The Street Collector. The platform serves three user types: collectors (buyers), vendors (artists/galleries), and admins.

The technical stack is Next.js (App Router) on the frontend, Supabase (Postgres + Auth + Edge Functions) on the backend, Shopify (headless) for commerce, Web NFC API for tag interaction, and Shopify Polaris Web Components for admin/vendor UI.

The README enumerates major feature areas: digital art management, NFC authentication, giveaway roulette wheel, payout system (Gumroad-level with PayPal/Stripe/Bank Transfer), collector profiles, and an Apollo-grade CRM.

## Key Takeaways

- The platform's core value proposition is NFC-linked digital certificates for physical artworks.
- Collector profiles are holistic — they aggregate Shopify orders, warehouse data, edition assignments, and activity history.
- The giveaway roulette uses GSAP animations and supports weighted entries (multiple NFC tags = multiple chances).
- The payout system supports multi-currency (GBP, EUR, CAD, AUD, NIS) with real-time exchange rates.
- NFC pairing requires Chrome, Edge, or Opera — Safari is not supported.
- The CRM is described as "Apollo-grade foundations" with Attio-style filters.
- Supabase Auth handles vendor/admin login; Shopify customer cookies handle collector auth.

## New Information

- GSAP is used for the giveaway roulette wheel animation.
- Payout scheduling supports bi-weekly, weekly, and monthly cycles.
- Instant payouts on-demand are supported with configurable fees.
- The collector dashboard includes an "artist journey" section.
